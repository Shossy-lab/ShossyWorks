# Feature Backlog

Track feature requests, friction points, and improvement opportunities discovered during sessions.

## Priority Levels
- **P0**: Blocking current workflow
- **P1**: Significant efficiency gain
- **P2**: Nice to have
- **P3**: Long-term / exploratory

## Categories
Tags: `[endpoint]`, `[tool]`, `[workflow]`, `[infra]`, `[ui]`, `[data]`, `[dx]`, `[security]`

## Trigger Conditions
Log a feature IMMEDIATELY when ANY of these occur:
- Workaround required for something that should be straightforward
- Missing capability that the user asks about
- Slow/multi-step process that could be simplified
- Manual step that could be automated
- User explicitly says "we should add X" or "it would be nice if..."
- Recurring error or limitation
- Friction point that slows development
