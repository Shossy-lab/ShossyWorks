# Feature Documentation

This directory contains:
- `backlog.md` — Feature requests and friction points (append-only, logged immediately during sessions)
- Individual feature detail docs as needed (created when a feature needs design discussion)

## Guidelines
- Keep individual feature docs under 5KB
- If a doc exceeds 10KB, split it into focused files
- Feature proposals go in their own file: `{feature-slug}.md`
- Reference contracts when features interact with other systems
