## Discoveries
- Vercel project was configured with Framework Preset "Other" instead of "Next.js" — caused 404 on all routes. Fixed by changing to Next.js in dashboard settings.
- Vercel Deployment Protection was enabled, blocking all visitors with 404. Disabled for production.
- Next.js 16 deprecates `middleware.ts` in favor of `proxy.ts` — still works, cosmetic warning only
- `@supabase/ssr` cookie methods need explicit typing (`CookieOptions`) in strict TypeScript
- Supabase is migrating from anon/service_role keys to publishable/secret keys (deadline late 2026) — code supports both

## Known Issues

- **CRLF warnings on git add** — Windows line ending normalization. Not blocking, but `.gitattributes` may need updating.
